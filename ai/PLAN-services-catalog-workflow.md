# Plan: Add or Update a Commercial Service Listing

This checklist is used to maintain the versioned `Services` pages generated from `data/services.csv`.

## Goal

Maintain a reliable and neutral catalog of commercial services that support OCPI.

## Data Source

- Single source of truth: `data/services.csv`
- One row per service.
- Public pages are generated by `tools/generate-antora-components.js`.

## CSV Schema

Required columns:

1. `company_name`
2. `service_name`
3. `category`
4. `website_url`
5. `product_url`
6. `catalog_versions`
7. `ocpi_versions`
8. `roles_supported`
9. `geographic_coverage`
10. `target_customers`
11. `notes`
12. `sponsorship_tier`
13. `source_urls`
14. `last_verified_date`
15. `active`

## Field Conventions

- Public site text stays in English.
- `service_name` must include the company name (for example `Plugsurfing Roam`, not only `Roam`).
- Avoid generic words in `service_name` such as `Platform`, `Solution`, `OCPI`, unless they are part of the official commercial product name.
- Use `|` as separator for multi-value fields:
  - `catalog_versions`
  - `ocpi_versions` (if multiple explicit versions are known)
  - `roles_supported`
  - `source_urls`
- `catalog_versions` accepts `all` or an explicit list like `2.3.0|2.2.1`.
- Allowed `category` values:
  - `Roaming Hub`
  - `CPO Platform/CPMS`
  - `eMSP Platform`
  - `OCPI Gateway/Integration`
  - `Interoperability Service`
  - `Other`
- `roles_supported` should use OCPI role labels when possible: `CPO|eMSP|Hub|NSP|SCSP|PTP|PSP`.
- `sponsorship_tier` values: `none|standard|featured`.
- `active` values: `yes|no`.
- If information cannot be verified, use `Unknown`.

## Workflow (Input: a URL)

1. Open the provided URL and identify the company, service, and product scope.
2. Confirm OCPI support from primary sources (official product/docs pages).
3. Collect factual metadata for all CSV fields.
4. Add or update a row in `data/services.csv`.
5. Set `last_verified_date` to current date (`YYYY-MM-DD`).
6. Keep `notes` short, factual, and neutral (1 sentence).
7. Regenerate Antora components:

   ```bash
   npm run generate:antora
   ```

8. Verify generated output:
   - `antora/components/ocpi-<version>/modules/ROOT/pages/services.adoc`
   - `antora/components/ocpi-<version>/modules/ROOT/nav.adoc`

9. Optional full build verification:

   ```bash
   npm run build:antora
   ```

10. If navigation/workflow behavior changed, update `README.md` and `AGENTS.md`.
